One player controlled the Tetris game, while the other controlled a physics-bound character, trying to dodge the tetronimos as they fell down. For this project, I worked on the input controls and physics engine of the game, and we ended up winning the competition with it.

TheCloudMC Minecraft Server

In 2014, a friend and I created our own Minecraft server; however, unlike the vast majority of people, I wanted to make it "Vanilla". This is a server with the original JAR provided by the developers, Mojang, with no external plugins enabled. My challenge was to get a server as functioning as any other plugin-based server through the use of in-game commands and command blocks.

We were able to accomplish all of this and more; I developed systems to set multiple homes, teleport request to other players, enable spawn protections with safe zones, moderate rule breakers, and much more. Because of all these features, we became the most popular Vanilla server in the world for months, and remain one of the more notable Vanilla servers to this day.

One of the largest challenges for this was ensuring the command blocks were optimized. Some features that normally wouldn't be a challenge when writing Java-based plugins can become very lag-inducive. This becomes more apparent due to the Mojang-provided JAR's poor multiplayer optimizations in general.

Discord Bot Manager

I started my creation of Discord bots in 2017 when I developed two different bots: one for a friend, and one for a YouTuber. These bots shared gambling commands, but had custom commands for each demographic (involving Pokemon, Clash Royale, and CS:GO).

As time progressed, and more bots were made, I started making the program simplified and expandable. This ended up being a more straightforward abstraction of the JDA library (which has helped my friend in developing his own Discord project).

My most popular bot is made to fetch speedrun data from Speedrun.com for both leaderboards and individual runs; their API, however, isn't always optimal, so I additionally used Selenium and JSoup to scrape info from their site's HTML. My latest bot is a group project manager which I initially created for my own use in my ongoing Faithfall project. It manages task organization (similar to JIRA) for the Discord platform and provides GitHub PR notifications.

Encryption

Plain Sight

For a local hackathon in Fall of 2017, I wanted to figure out how to hide text inside of any image you gave it with password protection. Two friends of mine helped in development of this 24-hour challenge where we magnified the image size and stored bits of data from slight offsets in its RGB values.

We won the competition but we weren't proud of the outcome; a year later, we improved upon the original design and placed Top 10 at the 2018 HackOHI/O OSU hackathon. I was still unhappy with how slow the finished version was; the GitHub repo shows a complete redesign by me, focusing on optimizations, securities, and long-term scalability.

Image Encryption

A predecessor to the above project, my intrigue with encryption in July of 2017 started with this concept. This simpler idea would, provided a password and text, create an image filled with noise (example shown). While this type of obfuscation may be more obvious than more, it does the job for simpler things.

One thing to improve upon for this project would be to randomize which pixels held your data; as it currently stands, it saves the encrypted data like one would a book, all pixels after an extent are random nonsense. Spreading out the data based on the password would be a much wiser decision moving forward.

Shuffull

A pet peeve of mine has been the poor quality of music shufflers; Spotify playlists replays songs too quickly and YouTube playlists don't properly shuffle playlists over 100 videos. This is the problem that I aspired to solve in May of 2017. Using a GUI, this program allows you to manage replayability of individual playlists locally.

Over the years, I have re-designed this program, and currently have plans to create a server-based version. This will allow for accounts to sync playlist data between multiple devices.

Sudoku Generator

Puzzle games have always been interesting for me, so I wanted to learn how to generate sudoku puzzles. After jokingly trying to randomly brute-force sudoku points (to no avail), I created a wholly unique algorithm in 2018. Along with normal sudokus, I also created an option to force-enable maps with diagonal matches. I was able to get the normal generation time down to ~10ms per map, and around 1-2 seconds per diagonal map.
